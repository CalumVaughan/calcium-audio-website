DroneStation

A web-based drone sound design explorer for algorithmic textures, delay networks, reverb spaces, and distortion.

Launch DroneStation

Description

Dronestation was my first RNBO-to-web project: an interactive browser-based sound instrument designed for exploring algorithmically generated drone textures, delay networks, reverb spaces, and distortion without relying on a traditional knob-driven interface.

The sound engine was built in GenExpr within RNBO, using five layered synthesiser voices routed through a shared procedural effects chain. The system generates rich, evolving drone states that continuously shift in texture, density, and harmonic character.

Rather than exposing a static interface, Dronestation is controlled entirely through movement. Mouse position maps directly onto the instrument's internal behaviour: the X-axis influences pitch behaviour, stereo width, delay mix, and generative motion, while the Y-axis controls brightness, reverb space, and damping. Clicking and dragging pushes the system into more unstable states, increasing density, drive, feedback, and temporal complexity.

This creates an instrument that remains active and responsive even when left untouched, blurring the line between performance and autonomous generative system.

These same audio parameters are mirrored into a procedural visual environment built using React, HTML Canvas, and WebGL. Delay, reverb, distortion, density, brightness, and pointer position all shape an abstract field of reactive geometry, pulsing forms, echo trails, and shifting spatial depth.

The visual language draws heavily from my previous Jitter work, reimagined for the browser as a unified audiovisual instrument.
